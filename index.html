# Defence & Athletics — Client Progress CRM A web app for tracking client plans, levels, and striking-skill progress. Data is stored in a shared **Supabase** database, so all coaches see the same roster live across every device. The app is a single `index.html` hosted free on GitHub Pages. --- ## What it does - **Dashboard** — active clients, sessions remaining, average skill mastery, and a "plans running low" alert. - **Clients roster** — searchable, filterable by level, with plan-usage and mastery bars. - **Client profile** — contact details, level (Beginner / Intermediate / Advanced), plan usage with one-tap session logging, ten striking skill areas rated 0–5, coach notes, and a dated progress log. Coaches enter a shared password to get in. Changes save to the shared database and appear on other devices within about 12 seconds. --- ## How it's wired - **Front end:** this `index.html` on GitHub Pages. - **Database:** a Supabase project with one table, `roster` (one row per client). - **Access:** a single shared password (a basic gate), plus Supabase's public "publishable" key, which is designed to be safe in browser code. --- ## Updating the app Edit `index.html`, then re-upload it to the GitHub repo (Add file → Upload files → Commit). GitHub Pages redeploys within a minute. Key things inside the file, near the top of the script: - `SUPABASE_URL` / `SUPABASE_KEY` — the database connection (already set). - `APP_PASSWORD` — change this to rotate the coach password. - `SKILL_AREAS` — add or rename the tracked techniques. - `PLAN_TYPES` — edit the available membership plans. --- ## Important to know - **Shared password = a gate, not strong security.** Anyone with the link and password can use it. Keep genuinely sensitive data (medical, payment) OUT of the system. For stronger protection you'd need per-coach logins and database rules — a larger upgrade. - **Needs internet.** The app loads React/fonts from a CDN and talks to the database over the internet. - **Free Supabase tier** pauses a project after about a week of zero activity; daily use keeps it awake. If it ever seems "asleep," opening the Supabase dashboard wakes it. - **Backups:** Supabase keeps your data, but consider periodically exporting the `roster` table from the Supabase dashboard for safety.
